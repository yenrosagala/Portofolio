Parameter Estimation With Acceptence Rejection Sampling¶

Acceptance-rejection sampling is a Monte Carlo method used to generate random numbers from a probability distribution. It works by generating random numbers from a simpler distribution (called the proposal distribution) and then accepting or rejecting them based on a certain criterion.

Steps:¶

Choose a proposal distribution: This distribution should be easy to sample from and should envelop the target distribution. Determine a constant M: This constant is the maximum value of the ratio of the target distribution to the proposal distribution. Generate random numbers: Generate random numbers from the proposal distribution. Accept or reject: For each generated number, calculate the ratio of the target distribution to the proposal distribution. If this ratio is less than or equal to M, accept the number; otherwise, reject it.

Advantages:¶

Simple to implement Can be used for complex distributions

Disadvantages:¶

Can be inefficient if the proposal distribution is a poor match for the target distribution Requires knowledge of the maximum value of the ratio of the target distribution to the proposal distribution

Applications:¶

Generating random numbers from complex distributions Simulation of physical systems Bayesian inference

Fungsi membangkitkan nilai parameter¶

Para prosedur accept-reject dengan tiga parameter seperti pada contoh ini adalah regresi linear berganda dengan parameter b0, b1, dan sigma, diperlukan prosedur estimasi nilai parameter yang memaksimumkan nilai log_likelihood, namun proses ini tidak dilakukan secara langsung, melainkan dengan menggunakan markov-chain. pada masing-masing fungsi genereta parameter, diberikan input nilai bagi parameter lain serta nilai data X dan Y, maka dengan funsi ini akan memperoleh nilai parameter yang memberikan nilai log_likelihood terbesar ketika parameter lain dan data telah diketahui. Hasil dari masing-masing parameter yang digenerate kemudian disimpan pada list yang bersesuaian.

from scipy.optimize import minimize

def log_likelihood(b0, b1, sigma): 
    lnL = (-(float(n)*0.5) * np.log(2.0 * np.pi * (float(sigma)**2.0))) - (0.5 * np.sum((y - float(b0) - float(b1)*x)/float(sigma))**2.0)
    return - lnL

def gen_b0():
    b0 = [b0s[-1]] #initial value dari componen terakhir list b0s
    b1 = b1s[-1]  #last value parameter b1
    sigma = sigmas[-1] #last value parameter sigma
    Results = minimize(log_likelihood, 
                           b0, 
                           method=Method,args=(b1, sigma),
                           options={'xatol': 1e-8, 'disp': False}
                           )
    b0s.append(Results.x[0])
    return Results.x[0]

def gen_b1():
    b1 = [b1s[-1]] #initial value dari componen terakhir list b1s
    b0 = b0s[-1]  #last value parameter b0
    sigma = sigmas[-1]  #last value parameter sigma
    Results = minimize(log_likelihood, 
                           b1, 
                           method=Method,args=(b0, sigma),
                           options={'xatol': 1e-8, 'disp': False}
                           )
    b1s.append(Results.x[0])
    return Results.x[0]

def gen_sigma():
    sigma0 = [sigmas[-1]]  #initial value dari componen terakhir list sigmas
    b1 = b1s[-1] #last value parameter b1
    b0 = b0s[-1]  #last value parameter b0
    Results = minimize(log_likelihood, 
                           sigma0, 
                           method=Method,args=(b0, b1),
                           options={'xatol': 1e-8, 'disp': False}
                           )
    sigmas.append(Results.x[0])
    return Results.x[0]

Fungsi Accept Reject Sampling¶

Parameter yang digenerate diterima sebagai bagian dari sampling jika nilai dari nilai likelihood dibandingkan dengan likelihood maksimum lebih besar dari angka random yang digenerate dari distribusi normal standar (0,1).

def accept():
    
    a = gen_b0()
    b = gen_b1()
    c = gen_sigma()
    
    u = np.random.normal(0, 1)
    value = log_likelihood(a,b,c) / max()
    if u < value:
        return {'b0': a , 'b1': b, 'sigma': c}

Fungsi Iterasi¶

Percobaan ini diulang sebanyak n iterasi, dimana setiap iterasi yang memperoleh parameter yang diterima, disimpan dalam array result. sedangkan parameter yang tidak diterima tidak disimpan. masing-masing parameter disimpan dalam sebuah list dimana data pertama dalam list merupakan initial guess untuk melakukan generate parameter.
